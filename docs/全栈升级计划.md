## 阶段一：项目工程化改造

> **预估时间**：1–2 小时 | **难度**：⭐ 简单

### Step 1 — 创建 Maven 多模块项目结构

1. 在项目根目录创建根 `pom.xml`，定义三个子模块：`search-engine-core`、`search-engine-server`、`search-engine-web`
2. 创建 `.gitignore`，添加 Java、Node.js、IDE 相关的忽略规则
3. 删除旧的不需要的文件：`bin/`、`SearchEngineForStudent.iml`、`index/index.dat`、`index/index.txt`、`index/search_result.txt`

### Step 2 — 迁移现有代码到 core 模块

| 操作 | 说明 |
|------|------|
| 移动 `src/` → `search-engine-core/src/main/java/` | 包名 `hust.cs.javacourse.search` 保持不变 |
| 移动 `text/` → `search-engine-core/src/test/resources/text/` | 测试数据集归类到测试资源 |

### Step 3 — 修复已知代码问题

| 文件 | 问题 | 修复方式 |
|------|------|----------|

## 阶段二：Spring Boot 后端

> **预估时间**：4–6 小时 | **难度**：⭐⭐ 中等

### Step 4 — 搭建 Spring Boot 骨架

**Maven 依赖**（`search-engine-server/pom.xml`）：

```xml
<dependencies>
    <dependency>
        <groupId>hust.cs.javacourse</groupId>
        <artifactId>search-engine-core</artifactId>
        <version>${project.version}</version>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-validation</artifactId>
    </dependency>
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>
```

**配置文件**（`application.yml`）：

```yaml
server:
  port: 8080

search-engine:
  document-dir: ./data/documents/
  index-dir: ./data/index/
  index-file: index.dat
  upload-max-size: 10MB
  ignore-case: true
  term-filter-pattern: "[a-zA-Z]+"
  term-min-length: 3
  term-max-length: 20

spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
```

**启动类**（`SearchEngineApplication.java`）：启动时自动检查并加载已有索引，若不存在则创建空索引。

### Step 5 — Service 层设计

`IndexService` 封装核心引擎调用：

| 方法 | 功能 | 说明 |
|------|------|------|
| `init()` | 初始化索引 | 启动时从文件加载，不存在则建空索引 |
| `addDocument(MultipartFile)` | 添加文档 | 保存文件 → 解析 → 加入索引 → 序列化 |
| `deleteDocument(int docId)` | 删除文档 | 从索引移除 → 删除文件 → 全量重建 |
| `listDocuments()` | 列出文档 | 返回 docId、文件名、路径、词条数 |
| `search(String keyword, int page, int size)` | 搜索 | 单/双词搜索 → 分页返回命中结果 |
| `rebuildIndex()` | 重建索引 | 遍历文档目录全量重建 |

### Step 6 — REST API 设计

#### 文档管理

```
POST   /api/documents/upload
       请求：multipart/form-data，字段名 "file"
       响应：{ code: 200, data: { docId, docName, termCount } }

GET    /api/documents
       响应：{ code: 200, data: [ { docId, docName, docPath, termCount }, ... ] }

GET    /api/documents/{docId}
       响应：{ code: 200, data: { docId, docName, docPath, content, termCount } }

DELETE /api/documents/{docId}
       响应：{ code: 200, message: "deleted" }
```

#### 搜索

```
GET    /api/search?q=keyword
       单词搜索，按词频得分降序返回

GET    /api/search?q=word1+word2&mode=and
       双词布尔搜索，mode 可选 and | or

GET    /api/search?q=keyword&page=0&size=10
       分页搜索，默认每页 10 条

响应格式：
{
  "code": 200,
  "data": {
    "totalHits": 42,
    "page": 0,
    "size": 10,
    "hits": [
      {
        "docId": 3,
        "docName": "article.txt",
        "docPath": "/data/documents/article.txt",
        "score": 15.0,
        "termFreqMap": { "search": 10, "engine": 5 },
        "snippet": "...full-text <mark>search</mark> engine..."
      }
    ]
  }
}
```

#### 状态

```
GET    /api/status
       响应：{ code: 200, data: { totalDocs, totalTerms, indexSize } }
```

#### 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

### Step 7 — Controller 与辅助类

| 类 | 职责 |
|----|------|
| `SearchController` | 接收搜索请求 → 调用 `IndexService.search()` → 返回 JSON |
| `DocumentController` | 文件上传 / 列出 / 详情 / 删除 |
| `StatusController` | 索引统计信息 |
| `ApiResult<T>` | 统一响应包装类（code, message, data） |
| `SnippetUtil` | 提取命中词上下文并包裹 `<mark>` 标签高亮 |
| `GlobalExceptionHandler` | `@RestControllerAdvice` 全局异常处理 |
| `CorsConfig` | 允许前端 `localhost:5173` 跨域访问 |

## 阶段四：部署与开源

> **预估时间**：2–3 小时 | **难度**：⭐⭐ 中等

### Step 12 — Docker 化

**部署策略**：React 构建为静态文件 → 复制到 Spring Boot `static/` → 一个 fat jar 包含前后端。

**`Dockerfile`**：

```dockerfile
# 构建阶段
FROM maven:3.9-eclipse-temurin-17 AS build
COPY . /app
WORKDIR /app
RUN mvn clean package -DskipTests

# 运行阶段
FROM eclipse-temurin:17-jre
COPY --from=build /app/search-engine-server/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

**`docker-compose.yml`**：

```yaml
version: '3.8'
services:
  search-engine:
    build: .
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data    # 持久化文档和索引
    environment:
      - SEARCH_ENGINE_DOCUMENT_DIR=/app/data/documents/
      - SEARCH_ENGINE_INDEX_DIR=/app/data/index/
```

一键启动：
```bash
docker-compose up -d
# 浏览器访问 http://localhost:8080
```

## 关键决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 后端框架 | Spring Boot 3.x / Java 17 | 现代化，你已具备 Java 基础 |
| 前端框架 | React 18 + TypeScript | 最流行，生态丰富，学习资源多 |
| UI 组件库 | Ant Design 5 | 开箱即用，风格专业，中文支持好 |
| 前端构建 | Vite | 比 Webpack 快 10 倍，开发体验好 |
| 生产部署 | 前端静态文件打包进 Spring Boot | 单 jar 部署，无需 Nginx，运维简单 |
| 排序算法 | 词频打分 → 后续升级 TF-IDF | 先跑通全链路，再优化精度 |
| 跨域方案 | 开发：Vite 代理 / 生产：同源 | 最简方案，零额外配置 |
| 索引存储 | Java 序列化（`index.dat`） | 保持简单，文档量少时性能足够 |
| 增量索引 | 上传/删除后全量重建 | 文档量小，后续可优化 |

## 学习路线（React 相关）

方案 A 需要你学习 React。以下是最小必要知识清单：

| 优先级 | 技术 | 学习内容 | 预估时间 |
|--------|------|----------|----------|
| 🔴 必须 | React 基础 | JSX 语法、函数组件、Props | 2–3 小时 |
| 🔴 必须 | React Hooks | `useState`、`useEffect`、`useRef` | 2–3 小时 |
| 🔴 必须 | TypeScript | 类型注解、interface、泛型（你有 Java 基础，很快） | 1–2 小时 |
| 🟡 重要 | Ant Design | 常用组件：Layout、Table、Form、Input、Upload、Card、Modal | 边用边查 |
| 🟡 重要 | Axios | GET/POST 请求、拦截器、错误处理 | 30 分钟 |
| 🟢 了解 | React Router | `BrowserRouter`、`Route`、`useNavigate` | 30 分钟 |

> **总计**：约 1.5–2 天可上手写页面

## 后续扩展方向

|------|------|--------|
| **TF-IDF / BM25 排序** | 替换 SimpleSorter，科学打分 | 🟡 中 |
